**Elden Ring Wiki — MP09 Projecte Final**
**Justificació i Motivació**
L'Elden Ring és l'últim joc que vaig jugar abans de que les pràctiques i les classes em treiessin gran part del meu temps i s'ha convertit en el meu videojoc preferit. És per això que li he volgut donar un petit homenatge fent una web informativa sobre el seu 'lore' i els seus 'bosses' més forts que tant em van costar entendre i vèncer.

**Esquema d'Arquitectura**
L'aplicació segueix l'arquitectura MVC de .NET Core. La capa de Model conté les entitats Boss i LoreEntry, gestionades amb Entity Framework Core sobre SQLite, i els models per deserialitzar les respostes de l'API externa amb Newtonsoft.Json. La capa de Controlador conté tres controladors: HomeController, BossController i LoreController, cadascun amb múltiples accions que gestionen les consultes a la BD i les crides a l'API. La capa de Vista conté totes les pàgines Razor, que reben les dades dels controladors i les mostren a l'usuari amb HTML, CSS i JavaScript.

**Explicació del Codi**
Els models defineixen les entitats principals de l'aplicació. Boss conté tots els atributs d'un boss (nom, dificultat, punts de vida, ubicació, imatges, lore) i LoreEntry representa cada entrada de la cronologia. Els models d'API utilitzen atributs de Newtonsoft.Json per mapar correctament els camps del JSON extern als objectes C#.
Els controladors gestionen la lògica de l'aplicació. HomeController crida l'API externa en arrancar per mostrar un boss destacat aleatori i consulta la BD per al top 3 de dificultat. BossController gestiona el llistat, el detall individual (enriquint la imatge des de l'API), el rànquing i la pàgina de dades en viu de l'API. LoreController gestiona la cronologia completa, el detall de cada entrada i el filtrat per categories. Tots els controladors fan ús de crides asíncrones amb async/await i gestionen els errors per no bloquejar l'aplicació si l'API externa no respon.
Les vistes utilitzen sintaxi Razor per combinar HTML amb C#, els Tag Helpers d'ASP.NET per generar URLs i un layout compartit que inclou la navbar, el footer i el reproductor de música flotant. El JavaScript gestiona el reproductor d'àudio MP3, les animacions d'entrada per scroll i les barres de dificultat animades. El CSS utilitza variables personalitzades per mantenir la coherència visual del tema fosc daurat.
Pel que fa a l'accessibilitat, totes les imatges tenen atribut alt descriptiu, els elements interactius tenen aria-label, les barres de dificultat usen role="meter" amb els valors corresponents, hi ha un skip link per saltar la navegació i tots els elements de llista usen etiquetes semàntiques correctes, preparat per ser validat amb WAVE.

**Propostes de Millora i Noves Funcionalitats**
Una millora interessant seria afegir un botó de traducció automàtica per convertir les descripcions que arriben en anglès des de l'API al català o castellà, mantenint l'opció de veure l'original. Una altra millora important seria fer que la música no s'interrompés en canviar de pàgina, cosa que es podria aconseguir convertint l'aplicació en una SPA amb React o carregant el contingut de forma parcial sense recarregar el layout. També seria útil afegir una barra de cerca per filtrar bosses per nom o dificultat, un mapa interactiu de les Terres Entremig amb marcadors per a cada boss i un sistema de paginació per explorar tots els bosses disponibles a l'API, que en conté més de cent.
